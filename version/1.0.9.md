# 1.0.9

发布日期：2026-03-24

## 本次目标
- 保持现有配色与背景不变，重写帖子详情页 UI 为稳定响应式布局。
- 解决详情页可能出现的元素遮挡、正文区宽度不稳定问题。
- 按 shadcn/ui 设计思路复用现有组件轮子，不重复手搓基础组件。

## 轮子与设计依据
- 参考：`https://ui.shadcn.com/docs/installation`
- 本次继续复用项目内已接入的 shadcn 风格组件：`Card`、`CardHeader`、`CardContent`、`CardTitle`、`Badge`、`Button`。
- 图片素材使用：`src/public/kuromi/16x9/kv_krm.png`、`src/public/kuromi/1x1/illust_2.png`。

## 修改文件

### `src/components/PostDetail.tsx`
- 在不修改接口与业务逻辑（`/api/posts/:id` 拉取、错误处理、返回主页行为）的前提下，重构详情页结构：
  - 顶部 Hero 信息区（标题、作者、发布时间、当前用户、访客模式说明）
  - 主内容区 + 侧边信息区的响应式布局：
    - 小屏：单列顺序排布
    - 大屏：`主内容 + 侧栏` 双列
  - 正文区保留 Markdown 渲染能力，新增详情页专用类名 `post-detail-markdown`
  - 侧栏新增 Meta 卡片、Tag 卡片与库洛米插画卡片
- 视觉上保持原有配色体系与背景氛围，提升可读性与层次。

### `src/index.css`
- 增加详情页专用覆盖，避免全局 `65%` 宽度在详情页触发错位：
  - `.post-detail-markdown.kuromi-markdown .markdown-body { width: 100%; max-width: none; }`
- 增加中小屏保护：
  - `@media (max-width: 1024px) { .kuromi-markdown .markdown-body { width: 100%; } }`
- 作用：不破坏其他页面视觉的同时，修复详情页正文宽度与遮挡风险。
