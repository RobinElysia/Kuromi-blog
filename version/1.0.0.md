# 1.0.0

## 修改目标
- 保持原有配色与背景体系。
- 新增首次打开加载页，加载完成后进入登录页。
- Home改为左中右三栏布局。
- 新增帖子详情独立页面（可从Home进入）。
- 修复前端中文乱码与文案不一致问题。

## 修改文件与内容

### src/App.tsx
- 新增首次访问加载逻辑：`kuromi_seen_intro_loading`（localStorage）。
- 新增轻量路由逻辑（基于 `history` + `pathname`）：
  - `/` -> Home
  - `/posts/:id` -> 帖子详情页
- 接入新组件：`LoadingScreen`、`PostDetail`。
- 将帖子打开行为从Home透传为 `onOpenPost`，支持详情跳转。

### src/components/LoadingScreen.tsx（新增）
- 新增首次访问加载页组件。
- 使用进度条与品牌图（`logo.svg`）展示启动状态。

### src/components/Login.tsx
- 修复全部中文乱码。
- 明确业务规则文案：仅 RobinElysia / Meow 可管理登录、无注册、支持访客模式。
- 账号输入登录时执行 `trim()`，减少误输入导致的登录失败。

### src/components/Home.tsx
- 全量重构为左中右三栏布局：
  - 左侧：头像、个人简介、建站时间（3.8）、运行时间、操作按钮。
  - 中部：发帖区 + 帖子流 + 评论区 + 详情入口。
  - 右侧：可点击时间轴，按帖子时间展示。
- 保留原有能力：发帖、图片上传压缩、评论、切换壁纸、退出登录。
- 修复全部中文乱码，统一文案语义。
- 新增站点运行时长计算（基于 `2026-03-08`）。

### src/components/PostDetail.tsx（新增）
- 新增帖子详情页（独立页面体验）。
- 展示帖子作者、发布时间、正文、图片、评论列表。
- 支持返回Home。

### server.ts
- 新增 `GET /api/posts/:id` 单帖详情接口，返回帖子及评论。
- 登录接口补充基础安全/健壮性处理：
  - 对 `username` 做 `trim()`。
  - 空账号/空密码返回 `400`。

## 质量审阅结果
- 编码乱码：已修复前端页面中的中文乱码问题，统一 UTF-8 可读文本。
- 安全性：
  - 已做基础输入校验与空值拦截。
  - 仍存在硬编码管理员密码（当前方案延续原设计），后续建议迁移到环境变量并加哈希存储。
- 最佳实践：
  - 组件职责清晰（Loading / Home / Detail 分离）。
  - 路由采用轻量实现，避免新增依赖，适合当前体量。
- 可读性：命名语义明确，页面结构分区清晰。
- 规整性：改动集中在 `App + components + server`，未引入多余目录与无关依赖。
- 冗余：复用原有发帖/评论逻辑，未重复造轮子。
