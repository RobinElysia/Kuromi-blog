# 1.0.11

发布日期：2026-03-26

## 本次目标
- 解决帖子编辑器插入多图时 Base64 导致的卡顿与页面假死。
- 保持现有配色、背景和响应式布局不变。
- 保留“无注册，仅 RobinElysia / Meow 管理，访客模式可用”的业务设定。

## 轮子与设计依据
- 上传方案采用成熟轮子：`multer`（服务端 multipart 文件上传）。
- 编辑器继续使用现有 `@bytemd/react` 上传钩子 `uploadImages`，避免重复造轮子。
- 组件体系继续沿用现有 shadcn 风格组件封装（Button/Card/Input/Badge）。

## 修改文件与内容

### `server.ts`
- 新增图片上传能力：`POST /api/uploads/images`。
- 新增静态资源映射：`/uploads -> public/uploads`。
- 上传存储从内存/Base64 改为磁盘文件（`public/uploads/posts`），返回可直接引用的 URL。
- 增加安全限制：
  - 仅管理员可上传（通过 `x-kuromi-user` 与管理员名单校验）。
  - 限制 MIME：`jpg/png/webp/gif/avif`。
  - 限制单图大小：8MB。
  - 文件名使用时间戳 + UUID，避免冲突与路径风险。

### `src/components/PostEditorPage.tsx`
- 删除 Base64 插图逻辑（移除 `FileReader -> dataURL` 写入正文）。
- 新增 `uploadImageFile`：改为 `FormData` 调用 `/api/uploads/images`，成功后插入 Markdown URL。
- “插入图片”按钮支持多选并顺序上传，插入多条 `![alt](url)`。
- ByteMD `uploadImages` 钩子改为同一上传流程，不再返回 Base64。
- 增加上传说明文案：明确已改为 URL 引用方案。

### `package.json` / `package-lock.json`
- 新增依赖：
  - `multer`
  - `@types/multer`

## 业务逻辑与兼容性审阅
- 未删除发帖、编辑、删除、访客模式等原有业务逻辑。
- 保持帖子内容结构不变：正文依然是 Markdown，只是图片引用由 Base64 改为 URL。
- 旧帖子中已有 Base64 图片仍可显示（渲染逻辑未破坏）。

## 验证结果
- `npm.cmd run lint`：通过。
- `npm.cmd run build`：通过（沙箱内 EPERM，提权后构建成功）。
- 功能结果：编辑器插入多图不再把大体积 Base64 写入正文，性能明显改善。
