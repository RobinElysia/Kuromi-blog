# 1.0.8

发布日期：2026-03-24

## 本次目标
- 继续修复帖子详情页正文仍有“文字遮挡/宽度不对齐”问题。
- 让详情页 Markdown 渲染区域和父容器宽度逻辑一致，不再出现正文被压缩或被裁切。

## 问题根因
- ByteMD Viewer 默认样式中：
  - `.bytemd` 有固定高度（300px）
  - `.bytemd-preview` 默认是 `inline-block`
- 在详情页中会导致预览区按内容宽度收缩，不稳定地出现“正文区变窄、右侧看似遮挡”的现象。

## 修改文件

### `src/index.css`
- 针对详情渲染容器 `.kuromi-markdown` 增加 Viewer 级覆盖：
  - `.kuromi-markdown .bytemd`
    - `height: auto;`
    - `overflow: visible;`
  - `.kuromi-markdown .bytemd-body`
    - `height: auto;`
    - `overflow: visible;`
  - `.kuromi-markdown .bytemd-preview`
    - `display: block;`
    - `width: 100%;`
    - `max-width: none;`
    - `height: auto;`
    - `overflow: visible;`
  - `.kuromi-markdown .markdown-body`
    - `width: 100%;`
    - `max-width: none;`
- 结果：正文区域宽度与父级内容区一致，不再被“内容宽度收缩”影响。

## 兼容性说明
- 不涉及 API、权限、数据结构改动。
- 仅修复详情页渲染容器布局，不影响编辑器能力（Markdown/LaTeX/Mermaid/图片）。
